Introduction to Function, Execution Context and the Call Stack

The moment we start running our code, JavaScrip creates the Global Execution Context

JavaScript execut the code line by line (parses)

Parsing and execution the code line by line is called thread of execution.

Code example:

                  1  const num =3;
                  2  function multiplyBy2 (inputNumber){
                  3      const result = inputNumber * 2;
                  4      return result;
                  5  }
                  6  const name = "will";

JavaScript is going to read line 1 (store variable), then line 2 and store the funciton in memory and jump to line 6.

If we don't invoke the function it will never read what is inside of it.(

This code is going to be available anywhere inside of our application, because this is the global excecution context.

JavaScript is going to go from top to bottom of our code, that means it is Synchronous. And because is excecution one thing at a time we said that JavaScrip is single Threaded

If we call the function: 

                    const output = multiplyBy2(4);
                    const newOutput = multiplyBy2(10)

When we execute a function we create a new execution context = local excecution context

In the Global execution context the function variable is going to receive the value of undefine.(output)

In the local execution context the first thing that is going to store in memory is going to be the parameter inputNumber with its argument that is number 4

After reading line by line the function is going to return the value of 8 to the global exectution context, and change the value of undefine to 8.

The way that JavaScrip has to track where is in the code or where is in the execution context is the Call Stack. Always at the bottom of the call stack is going to be our global excecution context

Then start to push on the function and when it finish with the function it pop off from the call stack. (first in last out)


Callbacks and Higher Order Functions

We have to create functions that permit us to reuse it as many times as possible.

We don't want to determine, or we don't want to be specific on until we end up running the function

Every time we invoke a new function JavaScritp will create a new execution context

Functions in JavaScript are first class objects. They can be treated like any other object

  • Assigned to variables and properties of other objects.
  • Passed as arguments into functions.
  • Returned as value from functions.

Functions have one property that objects don't have, and that is that functions can be invoke.

Higher order functions are the one that can handle other functions being passed in and returned out.

The function that is pass in is the CallBack Function.

Both allow us to run asynchronous code.
