Tools:

  • Slack: Most developers uses.
  • Trello: free tool. Can create a board with backlog, defects, doing and done.
  • GitHub: Is to share what they have done with each other. This tool track changes.

Putting all together

  1. Login to slack or teams a look for updates
  2. Open trello and look the Backlog and Doing list
  3. Pick up a task to do and move it to Doing
  4. Make a pull
  5. Open my editor and complete task.
  6. Push and Commit
  7. Move task in trello to Done
  8. Make any updates to Slack or Teams.

Managing The Modern Frontend Workflow

Web development projects are complicated. Some tasks that could need to happen every time something changes:

  1. Lint CSS and JavaScript
  2. Run Unit Tests to make sure nothing got broken
  3. Compile all SCSS/LESS to CSS
  4. Concatenate all CSS into one file for faster loading
  5. Minify the CSS to reduce size
  6. Transpile JS with Babel for support in older browsers
  7. Concatenate all JS file for faster loading
  8. Minify and Uglify JS to reduce size.
  9. Do the same with 3rd party CSS or JS
  10. Move all production assets into a distribution directory to separate from development part.

Tools fall into three categories:

  1. Package managers: We use npm for our package manager. Track of all the external dependencies for our app.
  2. Bundlers: Handle the compiling, transpiling, concatenating, minifying. We use SNOWPACK.
  3. Task Managers: These keep track of what needs to be done and when. We use npm.

GIT and GitHub

Version control is a method of recording changes made to a file or a set of files. Allow us to come back to a specific version.
Two types of control version: centralized and distributed (git).
To configure:

  • git config --global user.name “user name”
  • git config --global user.email “email”

3 Files stages in Git: Modified (working directory), Staged (staging area) and Committed (repository).
Git commands to navigate through the 3 stages: git add, git commit, git push.
The basic workflow is:

  1. Modified files in the working directory
  2. add files to the staging area
  3. commit files to the repository

Commands:

  • git remote add origin http://repository address
  • git remote -v
  • git status
  • git add . or git add filename
  • git commit -m “message for the commit”
  • git push origin master ==== to push the changes to the repository

Commands for the log:

  • git log
  • git log --pretty:oneline ==== to see the commit history in one lines
  • git log --author = “usuario” ==== to see the log for that user
  • git show “number of merge” ==== shows the merge with its changes.
  • git diff “branchName” “branchName”==== shows the difference.

Commands to undo mistakes:

  • git reset Head “filename” === is going to taken off for the stage
  • git reset --soft Head~1 ==== undo something that you commit.
  • git revert HEAD ==== undo a push.

Commands when there is changes in the repository:

  • git fetch ==== will fetch changes for the remote repo.
  • git merge origin/master ==== performed a fast forward.

And easy way to fetch and merge remote chenges into local repo: git pull origin master



Branches

The main branch is called Master.
Git branches are independent with its own unique changes.
Head refers to the tip of the branch you are on.
When create a branch it has all the code that is in master (is a copy of master)
Commands:

  • git branch -a ==== shows all the current branches.
  • git branch “branchName” ==== creates a branch
  • git checkout “branchName” ==== change of branch
  • git checkout -b “branchName” ==== creates and go to that branch
  • git branch -m “branchName” “newBranchName” ==== change branch Name
  • git branch -d “branchName” ==== delete a branch
  • git push origin “branchName” ==== push changes to a specific branch

Mergin Brances

Fast forward merge: if there is no development into the master branch (no diverged) Then the git Head will point to the latest commit on the branch after the merge.

No fast-forward merge: No development has taken place on the master branch. Instead of pointing to the last commit on the branch you are working on, a separate commit is represented to the merge.

Commands:

  • git merge --no-f “branchName” ==== no fast-forward merge.
  • git merge “branchName” ==== fast-forward merge.
  • git branch -d “branchName ==== delete a branch.
  • git push origin --delete “branchName” ==== delete a branch on the repository.

To clone a repository from github to our local working directory: git clone “http://repository address”
